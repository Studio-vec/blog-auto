[ 전체 흐름 ]
Claude API로 글 생성 → 티스토리 API로 발행 → 네이버 블로그 자동화 발행
주 2회(화·금) 오전 9시 자동 실행

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  playwright install chromium

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 3. Claude API 키 발급
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. https://console.anthropic.com 접속 (회원가입 필요)
2. 우측 상단 → API Keys → Create Key
3. 키 복사 (sk-ant-... 로 시작)
4. 월 $5 크레딧 무료 제공 (약 글 100~200편 분량)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 4. 티스토리 설정 (수익화 핵심)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【앱 등록】
1. https://www.tistory.com/guide/api/manage/register 접속
2. 서비스명, 설명 입력 → CallBack URL: https://www.tistory.com
3. App ID, Secret Key 복사

【Access Token 발급】
브라우저 주소창에 아래 URL 입력 (APP_ID 교체):
https://www.tistory.com/oauth/authorize?client_id=APP_ID&redirect_uri=https://www.tistory.com&response_type=token

→ 로그인 후 주소창에 access_token=xxxx 가 포함된 URL이 뜸 → 복사

【AdSense 연결】
1. 티스토리 관리 → 수익 → AdSense 연결
2. Google AdSense 계정 연결 (없으면 신규 신청)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 5. .env 파일 생성
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
blog 폴더에 .env 파일 생성 후 아래 내용 입력:

  ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
  TISTORY_APP_ID=발급받은_앱ID
  TISTORY_SECRET_KEY=발급받은_시크릿키
  TISTORY_ACCESS_TOKEN=발급받은_액세스토큰
  TISTORY_BLOG_NAME=내블로그이름  ← myblog.tistory.com 이면 myblog
  NAVER_ID=네이버아이디
  NAVER_PW=네이버비밀번호
  POST_DAYS=TUE,FRI
  POST_TIME=09:00

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 6. 테스트 실행
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 티스토리만 테스트
  python main.py --now --tistory

# 네이버만 테스트
  python main.py --now --naver

# 둘 다 테스트
  python main.py --now

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 7. 자동 실행 설정 (Windows 작업 스케줄러)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PowerShell (관리자 권한)에서 실행:

  $action = New-ScheduledTaskAction -Execute "python" `
    -Argument "C:\Users\kty12\OneDrive\Desktop\blog\main.py" `
    -WorkingDirectory "C:\Users\kty12\OneDrive\Desktop\blog"

  $trigger1 = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Tuesday -At 9:00AM
  $trigger2 = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Friday -At 9:00AM

  Register-ScheduledTask -TaskName "TravelBlogPost" `
    -Action $action -Trigger $trigger1,$trigger2 `
    -RunLevel Highest -Force

  ※ python main.py (--now 없이) 는 스케줄러 모드로 실행돼서
     작업 스케줄러 대신 항상 켜놓는 방식도 가능.
